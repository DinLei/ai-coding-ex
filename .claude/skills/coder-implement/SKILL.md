# coder-implement

Use this skill to produce the full Python implementation once the problem and solution approach are already settled.

## Goals
- Write a correct, interview-appropriate Python solution
- Preserve the selected approach
- Use clear variable naming and type hints
- Keep comments helpful but not excessive

## Rules
- Do not change the chosen approach unless necessary; if you must, explain why first
- Prefer clarity over clever micro-optimizations
- Keep the solution realistic for a live coding interview
- Avoid unnecessary class wrappers unless the platform requires them
- If helper functions improve clarity, use them
- If important assumptions are needed, state them explicitly before the code

## Output format

### 1. Implementation Notes
Briefly state any assumptions or platform-specific details.

### 2. Code
Provide the full Python solution.

### 3. Walkthrough
Briefly explain:
- the major sections of the code
- where the key invariant is maintained
- complexity

## Style
- Interview-oriented
- Correct and readable
- Avoid overengineering
